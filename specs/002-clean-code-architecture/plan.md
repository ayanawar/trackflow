# Implementation Plan: Clean Code Architecture Refactor

**Branch**: `003-clean-code-arch` | **Date**: 2026-05-13 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/002-clean-code-architecture/spec.md`

---

## Summary

Refactor the TrackFlow Next.js 14 application to enforce a repository → service → route-handler layering pattern across all API routes, extract shared React Query hooks for all data resources, fix active security vulnerabilities (browser-to-Anthropic direct calls, `dangerouslySetInnerHTML` XSS, hardcoded JWT secret fallback), add a server-side AI proxy with rate limiting, replace the `prompt()` tag input with an inline UI control, and consolidate all Zod schemas.

---

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode already enabled)
**Primary Dependencies**: Next.js 14.2.5 (App Router), Prisma 5.16, Zod 3.23, React Query 5.51, Zustand 4.5, jose 5.6, bcryptjs 2.4, Axios 1.7
**Storage**: PostgreSQL via Prisma (no schema changes)
**Testing**: No test runner configured; smoke testing via browser and `tsc --noEmit`
**Target Platform**: Web — Node.js server (Vercel)
**Performance Goals**: Stats API ≤4 DB queries; stats cache 30s TTL; AI proxy ≤10 req/min per user
**Constraints**: No DB schema changes; no breaking changes to existing API shapes; `strict: true` already on
**Scale/Scope**: Single-user workspaces; standard Vercel single-instance deployment

## Implementation Sequence

### Group 1 — Foundation (no breaking changes, done first)

These changes are safe to land first because they add new files or modify `lib/` utilities without touching route handlers or components.

**1a. JWT_SECRET guard** (`src/lib/auth.ts`)
- Add startup check: if `JWT_SECRET` is undefined or equals the known-insecure default in `production`, throw `Error('JWT_SECRET is not set')`. In `development`, `console.warn`.
- Remove the `|| 'trackflow-secret-key-change-in-production'` fallback from the `TextEncoder` call.

**1b. Centralize schemas** (`src/lib/schemas.ts`)
- Move `updateSchema` from `src/app/api/auth/me/route.ts` → `src/lib/schemas.ts` as `updateUserSchema`.
- Add `aiQuerySchema = z.object({ question: z.string().min(1).max(500), context: z.object({...}) })`.

**1c. Rate limiter** (`src/lib/rateLimit.ts`)
- Implement sliding window: `checkRateLimit(userId, limit, windowMs)` returns `{ allowed: boolean }`.

**1d. authStore logout decoupled** (`src/lib/authStore.ts`)
- Remove `window.location.href = '/auth/login'` from the `logout` action.
- `logout` only calls the API and clears state. Navigation handled in Sidebar and mobile header call sites.

**1e. Remove demo credentials** (`src/app/auth/login/page.tsx`)
- Delete the `<div>` containing `Demo: demo@trackflow.com / password`.

### Group 4 — Route Handler Migration (wire up services)

Each route handler is replaced with a thin version: authenticate → validate → call service → return. All 11 route files are updated in this group.

| Route file | Service called |
|---|---|
| `api/auth/login/route.ts` | `auth.service.login` |
| `api/auth/register/route.ts` | `auth.service.register` |
| `api/auth/me/route.ts` | `auth.service.getMe`, `auth.service.updateMe` |
| `api/auth/logout/route.ts` | *(no service needed — just clear cookie)* |
| `api/projects/route.ts` | `project.service.listProjects`, `project.service.createProject` |
| `api/projects/[id]/route.ts` | `project.service.updateProject`, `project.service.deleteProject` |
| `api/time-entries/route.ts` | `timeEntry.service.listEntries`, `timeEntry.service.createEntry` |
| `api/time-entries/[id]/route.ts` | `timeEntry.service.updateEntry`, `timeEntry.service.deleteEntry` |
